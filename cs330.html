Software Design & Engineering · CS-330

3D Scene — OpenGL Rendering Engine

An OpenGL 4.6 / C++17 3D scene enhanced from a base kitchen layout into a fully lit, Skyrim-inspired environment with modular architecture, memory-safe rendering, and real transparency effects.

Artifact Description

This artifact is a 3D rendering project originally built for CS-330: Computational Graphics and Visualization. Using OpenGL 4.6, C++17, GLFW, GLEW, and the GLM mathematics library, I constructed a scene of objects resting on a stone paver floor. The original version included a peanut can, mixing bowl, and syrup bottle. My enhancements added a five-part modeled wine glass (base cylinder, stem cylinder, stem knob sphere, tapered bowl, torus rim, and a transparent dark red wine sphere) and a five-part candle (wax cylinder, wick, tapered flame body, flame tip sphere, and a wide amber cylinder simulating a glowing floor effect).

Why I Selected This Artifact

I selected this project because it showcases a wide range of technical competencies relevant to software development: 3D mathematics for modeling and projection, shader programming, interactive camera systems, memory management, and debugging at the graphics driver level. These skills are transferable to game development, simulation, visualization tools, and any domain requiring real-time rendering.

The honest starting point also matters. I had to re-enter a codebase that had broken in the time since the original course — reacquiring dependencies, rebuilding the environment, and resolving initialization-order crashes. That process, not just the final result, demonstrates professional maturity.

Enhancements Implemented

  • Fixed critical memory leak: replaced incorrect glGenTextures call in the cleanup path with the correct glDeleteTextures
  • Implemented three-point lighting with warm and cool color mixing for more realistic shading on reflective surfaces
  • Added scroll-wheel zoom to the free-flying camera for intuitive scene navigation
  • Designed and built a five-component wine glass using hierarchical object composition
  • Engineered alpha blending with correct render order: opaque objects first, translucent objects (wine sphere, amber glow cylinder) rendered after blending is enabled
  • Modeled five-component candle with stacked geometry aligned using camera clipping plane adjustments to eliminate gaps
  • Added shader and texture path validation with descriptive error output on failure
  • Refactored code into modular files: SceneManager, ShaderManager, Camera — each with a single clear responsibility

Course Outcomes Addressed

Collaborative Environments

Modular file structure (SceneManager, ShaderManager, Camera) allows new contributors to extend the scene without touching unrelated systems.

Algorithmic Principles

Alpha blending render order and point-light attenuation formula demonstrate physics-based algorithmic reasoning, not arbitrary choices.

Innovative Techniques

Scroll-wheel zoom and the candle glow effect (a flat translucent cylinder) are simple but effective solutions that avoid expensive post-processing.

Security Mindset

All shader and texture file paths are validated at load time; failures produce explicit error messages so no issue goes silently unnoticed.

Reflection

The most important lesson from this enhancement was about initialization order. GLEW must be initialized after an OpenGL context exists; calling it before produced null function pointers that caused hard-to-trace crashes. This kind of system-level dependency isn't visible in the code itself — it requires understanding the runtime environment, which is exactly the kind of knowledge that separates working code from production-ready code.

Three-point lighting reinforced how much visual quality comes from color theory, not raw computation. Blending warm and cool light sources is the difference between a flat-looking scene and one that reads as believable. And transparency — specifically the wine glass — taught me that rendering order is not an implementation detail but a fundamental part of the algorithm.

Re-entering the project after a year away and making it run again was arguably the most professionally relevant part of the process.
